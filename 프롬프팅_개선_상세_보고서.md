---

## 1. 문제 상황 분석

### 발견된 문제

**예시 블록 (471-474):**
```
471: No, so in this query, because I'm querying for title and name.
472: Uh, this line 11 essentially.
473: Isn't doing anything aside from one important fact, which is.
474: Suppose there's an option in my data...
```

**문제점:**
1. 하나의 긴 문장이 여러 블록으로 쪼개짐
2. 각 블록이 온점(.)으로 끝나지만 실제로는 문장이 완결되지 않음
3. 번역 시 각 블록을 독립된 문장으로 취급하여 부자연스러운 번역 생성

**원인:**
- Whisper API가 화자의 음성 휴지(pause)를 문장 종결로 오인
- 억양 변화를 문장 끝으로 판단
- 구어체의 불완전한 문장 구조를 제대로 처리하지 못함

##### A. 새로운 패턴 추가

```python
# 1. 종속절 감지 패턴
_SUBORDINATE_START = re.compile(
    r'^\s*(because|since|although|though|while|if|when|where|unless|until|as|after|before)\b',
    re.I,
)
```
**목적**: `because`, `since`, `although` 등으로 시작하는 문장은 온점으로 끝나도 불완전한 문장으로 판단

**적용 예시**:
- "because I'm querying for title and name." → 다음 블록과 병합

---

```python
# 2. 관계절/보어절 감지 패턴
_INCOMPLETE_CLAUSE = re.compile(
    r'\b(which is|that is|who is|what is|where is|how is|why is)\s*[.!?,]?\s*$',
    re.I,
)
```
**목적**: `which is`, `that is` 등으로 끝나는 문장은 명백히 뒤에 내용이 이어져야 함

**적용 예시**:
- "aside from one important fact, which is." → 다음 블록과 병합

---

```python
# 3. 소문자/필러 시작 감지 패턴
_LOWERCASE_OR_FILLER_START = re.compile(
    r'^[a-z]|^(uh|um|ah|oh|er)',
    re.I,
)
```
**목적**: 다음 블록이 소문자나 필러로 시작하면 같은 문장으로 간주

**적용 예시**:
- 블록 A: "...name." [0.3초 갭]
- 블록 B: "uh, this line..." ← 소문자/필러로 시작
- → 같은 문장으로 병합

---

##### B. 시간 갭 기반 병합

```python
SHORT_GAP_MS = 500   # 0.5초 미만 갭은 같은 문장으로 간주

# 시간 갭 계산
if idx + 1 < len(cleaned):
    time_gap_ms = cleaned[idx + 1].start.ordinal - sub.end.ordinal

# 짧은 갭 + 소문자 시작 = 같은 문장
short_gap_with_lowercase = (time_gap_ms < SHORT_GAP_MS) and next_starts_lowercase
```

**목적**: 블록 간 시간 간격이 0.5초 미만이고 다음 블록이 소문자로 시작하면 같은 문장으로 병합

---

##### C. 종합 판단 로직

```python
# 불완전 문장 판단 (기존 + 신규)
is_incomplete = (
    is_dangling                    # 기존: 전치사/접속사로 끝남
    or is_short_no_end             # 기존: 짧고 문장 부호 없음
    or starts_with_subordinate     # 신규: 종속절로 시작
    or ends_with_incomplete_clause # 신규: 관계절로 끝남
    or short_gap_with_lowercase    # 신규: 짧은 갭 + 소문자
)
```

**결과**: 5가지 조건 중 하나라도 만족하면 다음 블록과 병합

## 6. 향후 개선 계획

### 6.1 단기 (1~2주)

1. **매우 긴 문장 처리**
   - 30초 이상 블록을 의미 단위로 분할
   - 호흡 단위 감지 알고리즘 추가

2. **중첩 구조 처리**
   - 괄호, 인용문 내부의 문장 구조 인식

### 6.2 중기 (3~4주)

1. **머신러닝 기반 문장 경계 감지**
   - 음성 특징(피치, 강도, 속도) 활용
   - BERT 기반 문장 완결성 판단

2. **화자 인식 통합**
   - Whisper의 화자 분리 기능 활용
   - 화자별 발화 패턴 학습

3. **사용자 피드백 반영**
