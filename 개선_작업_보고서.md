## 1. 작업 배경

### 발견된 문제

영상 자막 번역 과정에서 **문장이 부자연스럽게 끊기는 현상** 발견

**문제 사례**:
```
영어 원문:
- "because I'm querying for title and name."
- "Uh, this line 11 essentially."
- "Isn't doing anything aside from one important fact, which is."

한국어 번역:
- "제목과 이름을 쿼리하고 있기 때문에,"
- "이 11번째 줄은 본질적으로,"
- "중요한 사실 하나를 제외하고는 아무것도 하지 않습니다."
```

**문제점**: 실제로는 하나의 긴 문장인데, 여러 조각으로 쪼개져서 번역되어 부자연스러움

---

## 2. 원인 분석

### Whisper API의 전사 특성

1. **음성 휴지 감지**: 화자가 말하다가 잠깐 멈추면 문장이 끝난 것으로 판단
2. **억양 변화**: 목소리를 내리면 온점(.)을 찍음
3. **구어체 처리 한계**: 회의/강의 같은 즉흥 발화는 문장 구조가 불완전한 경우가 많음

**결과**: 하나의 문장이 여러 블록으로 쪼개지고, 각 블록이 온점으로 끝남

### 3.1 핵심 개선: 똑똑한 문장 병합 알고리즘

**수정 파일**: `transcriber.py` (Line 218~350)  
**수정 함수**: `_clean_whisper_artifacts`

#### 추가된 5가지 휴리스틱

| 번호 | 휴리스틱 | 설명 | 예시 |
|------|----------|------|------|
| 1 | **종속절 감지** | `because`, `since`, `although` 등으로 시작하는 문장은 온점이 있어도 불완전 | "because I'm querying..." → 병합 |
| 2 | **관계절 감지** | `which is`, `that is` 등으로 끝나는 문장은 뒤에 내용이 이어져야 함 | "...which is." → 병합 |
| 3 | **시간 갭 감지** | 블록 간 시간 간격이 0.5초 미만이면 같은 문장으로 간주 | [0.3초 갭] → 병합 |
| 4 | **소문자/필러 감지** | 다음 블록이 소문자나 "uh", "um"으로 시작하면 이어지는 문장 | "uh, this line..." → 병합 |
| 5 | **화자 전환 감지** | `But`으로 시작하는 긴 문장은 별도 블록으로 분할 | "But we want..." → 분할 |

#### 개선 전후 비교

**Before (개선 전)**:
```
블록 471: "because I'm querying for title and name."
블록 472: "Uh, this line 11 essentially."
블록 473: "Isn't doing anything aside from one important fact, which is."
블록 474: "Suppose there's an option..."

→ 4개의 독립된 문장으로 번역
```

**After (개선 후)**:
```

→ 1개의 자연스러운 문장으로 병합 후 번역
```

### 3.2 프롬프팅 전략 (기존 유지)

**관련 파일**: `prompts.py`, `translator.py`

#### A. 시스템 프롬프트 구조

GPT에게 "전문 영상 자막 번역가" 역할을 부여하고 12가지 규칙 전달:

```python
def build_system_prompt(target_lang, glossary_path):
    """
    GPT에게 전달하는 시스템 프롬프트 생성
    
    핵심 규칙:
    1. translate 배열만 번역 (context는 참고용)
    2. id 집합과 순서는 입력과 정확히 일치
    3. 한 블록 = 하나의 번역
    4. 의역 허용, 의미 기반 자연스러운 표현
    5. 회의/교육 톤 (~합니다/~입니다)
    6. 기술 용어는 영문 유지
    7. 고유명사/제품명은 원문 유지
    8. 원문 부호 구조 유지 (쉼표→마침표 변환 금지)
    9. 구어체 필러 제거 가능
    10. 줄바꿈은 자연스러운 경우만
    11. 용어집 자동 적용
    12. JSON 스키마 준수
    """
    return prompt_text
```

#### B. 번역 요청 페이로드

```python
def build_user_payload(translate_blocks, context_before, context_after, target_lang):
    """
    GPT에게 전달하는 번역 요청 데이터
    
    구조:
    {
        "target_language": "Korean",
        "context_before": [앞 5개 블록],  # 문맥 참고용
        "translate": [번역할 30개 블록],   # 실제 번역 대상
        "context_after": [뒤 5개 블록],    # 문맥 참고용
        "instruction": "번역 지시사항"
    }
    """
    return json.dumps(payload)
```

**특징**:
- 30개 블록씩 청크로 묶어서 번역 (효율성)
- 앞뒤 5개 블록을 문맥으로 제공 (자연스러움)
- 용어집(`glossary.json`) 자동 주입

#### C. 번역 안정성 보장

```python
class Translator:
    def translate_chunk(self, chunk):
        """
        청크 단위 번역 + 자동 재시도
        
        1단계: 전체 청크 번역 시도
        2단계: id 누락 시 해당 블록만 재요청
        3단계: 최종 실패 시 원문 유지
        
        안정성 메커니즘:
        - JSON Schema strict 모드 (id 매핑 보장)
        - Exponential backoff 재시도 (API 장애 대응)
        - 텍스트 정리 (줄바꿈/공백/숫자 제거)
        """
        # 1) 첫 시도
        got = self._call_api(chunk.translate, chunk.context_before, chunk.context_after)
        
        # 2) 누락된 id만 재시도
        missing = [i for i in expected_ids if i not in got]
        if missing:
            more = self._call_api(sub_blocks, context_before, context_after)
            got.update(more)
        
        # 3) 최종 결과 (누락 시 원문 유지)
        for i in expected_ids:
            result[i] = got.get(i, original_text[i])
        
        return result
```

## 7. 금일(5/8) 추가 개선 작업

### 7.1 화자 전환 패턴(`_SPEAKER_SPLIT`) 대폭 축소

**수정 파일**: `transcriber.py` (Line 117~122)

**문제**: 기존 정규식이 `but`, `I think`, `sure`, `that`, `or` 등 **일반 접속사/단어**에서도 문장을 분할하여, 한 문장이 3~6조각으로 분해되는 현상 발생

**개선 전** (위험한 패턴 다수 포함):
```python
_SPEAKER_SPLIT = re.compile(
    r'(?<=[\s.!?,])(?=(?:but |so just|just to|I think|I would|I was|'
    r'I see|I need|I don\'t|sure |that\'s a|that\'s how|and so|...))',
    re.I,
)
```

- `sure ` → "make **sure** that..." 에서 오분할
- `I think` → "what **I think** is important..." 에서 오분할
- `but ` → "I like it **but** I'm not sure" 에서 오분할
- lookbehind에 **공백(`\s`)** 포함 → 어디서든 발동

**개선 후** (명확한 응답어만):
```python
_SPEAKER_SPLIT = re.compile(
    r'(?<=[.!?,])\s+(?=(?:Yeah|Yes|No|Okay so|Thanks|Thank you|'
    r'That\'s great|Absolutely|Exactly|Right so|All right))',
    re.I,
)
```

- **쉼표/마침표 뒤에서만** 발동 (공백만으로는 불가)
- "Yeah", "Yes", "No" 등 **명확한 응답어**만 분할 대상
- 일반 접속사(`but`, `so`, `and`)는 분할하지 않음

### 7.2 문장 분할(`_SENT_SPLIT`) 정밀화

**문제**: `(?<=[.!?])\s+` → "2.5 million", "version 3.0" 등 숫자 뒤 마침표에서도 분할

**개선 후**:
```python
_SENT_SPLIT = re.compile(r'(?<=[.!?])\s+(?=[A-Z])')
```

- `.!?` 뒤에 **대문자**가 올 때만 분할 (진짜 새 문장)
- "2.5 million" → 분할 안 됨 (소문자 m)
- "...done. Next step..." → 분할됨 (대문자 N)
