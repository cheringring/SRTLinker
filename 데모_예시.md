- 이 상태로 번역하면 각 조각이 독립적으로 번역되어 부자연스러움

### [2단계] 영어 후처리 (en.srt)


분할된 블록을 문장 단위로 병합. 필러 제거, 화자 전환 분리 적용.

```
1
00:00:00,000 --> 00:00:08,360
We did recently switch our version of Teams, so I just need to find how it works now.

7
00:00:31,920 --> 00:00:46,919
So, before we jump in today, I want to take a minute to basically address a few questions that came up yesterday that we didn't get into or didn't have time to talk about.

11
00:00:56,919 --> 00:01:25,919
In order to basically create any sort of ticket, you can submit your tickets as getting IT help for now, since obviously we aren't actually doing development yet, but we want to make sure that any questions or issues that you're having, whether it's with how the trainings are going or with content that you feel is being missed, feel free to use this top box in order to create a ticket to get that concern heard.
```

**개선 결과:**
- 블록 1~2 → **블록 1** (1문장 = 1블록)
- 블록 8~11 → **블록 7** (4개 → 1개로 병합)
- 블록 16~23 → **블록 11** (8개 → 1개로 병합)
- 화자 전환 시 자동 분리 ("Can you all see?" → "Yeah, I can see that.")

### [3단계] GPT-4o 한국어 번역 (ko.srt)

완전한 문장 단위로 번역되어 자연스러운 한국어 자막 생성.

```
1
00:00:00,000 --> 00:00:08,360
최근에 Teams 버전을 변경했기 때문에, 이제 어떻게 작동하는지 알아봐야 합니다.

7
00:00:31,920 --> 00:00:46,919
오늘 시작하기 전에, 어제 나왔지만 다루지 못했거나 시간이 부족했던 몇 가지 질문을 잠시 다루고 싶습니다.

11
00:00:56,919 --> 00:01:25,919
기본적으로 티켓을 생성하려면, 현재 개발을 실제로 진행하고 있지 않기 때문에 IT 지원을 받는 것으로 티켓을 제출하실 수 있습니다, 하지만 교육 진행 방식이나 누락된 콘텐츠에 대한 질문이나 문제가 있다면 이 상단 박스를 사용하여 티켓을 생성해 주시면 됩니다.
```

**번역 품질:**
- 기술 용어(Teams, IT) 영문 유지
- 원문의 쉼표 구조를 번역에서도 유지 (한 문장 내 쉼표 → 번역에서도 쉼표)
- 경어체 일관 사용 (~합니다, ~됩니다)
- 영어/한국어 SRT 블록 수·타임스탬프 완전 일치 (1:1 매핑)

## 화자 전환 감지 예시

```
[영어]
4: Can you all see?              ← 발표자 질문
5: Yeah, I can see that.         ← 참석자 응답 (자동 분리)
6: Awesome.                      ← 발표자 반응

[한국어]
4: 모두 보이시나요?
5: 네, 보입니다.
6: 좋습니다.
```

- 질문(?) 뒤에 "Yeah"로 시작하는 응답을 자동으로 새 블록으로 분리

## 영어 후처리 로직 상세

### 처리 순서 (`transcriber.py` → `sentence_merger.py`)

```
Whisper raw.srt
  │
  ├─ [1] 긴 블록 분할 (_split_long_blocks)
  │     ├─ 8초 초과 블록 → 문장 종결 부호(. ? !)로 분할
  │     ├─ ? 뒤 답변 패턴 감지 → 화자 전환 분할
  │     └─ 짧은 블록도 여러 문장이면 분할
  │
  ├─ [2] 필러 제거 + 불완전 문장 병합 (_clean_whisper_artifacts)
  │     ├─ 필러 전용 블록 삭제 (Uh, Um, Hmm, Yeah, So, And, But 등)
  │     ├─ 마침표 없는 블록 = 불완전 문장 → 다음 블록과 병합
  │     └─ 병합 제한: 완결 문장 15초, 불완전 문장 30초
  │
  └─ [3] 문장 단위 병합 (sentence_merger.py)
        ├─ 문장 종결 부호로 그룹 종료
        ├─ 화자 전환 감지 (질문→응답, 시간 갭, 응답어 패턴)
        └─ 최대 8블록까지 하나로 병합
```

### 화자 전환 감지 기준 (`sentence_merger.py`)

| 기준 | 설명 | 예시 |
|------|------|------|
| 질문→응답 | 이전 블록이 `?`로 끝남 | "Can you see?" → "Yeah, I can." |
| 시간 갭 | 블록 간 1초 이상 공백 | 화자가 바뀌면 자연스러운 침묵 발생 |
| 응답 시작어 | Yeah, No, Sure, Okay, Thanks 등 | "Thanks, that's helpful." |

### 필러 제거 패턴 (`transcriber.py`)

```
삭제되는 필러 전용 블록:
uh, um, hmm, ah, oh, er, huh, mhm, okay, so, and, but, right, yeah, yes, well, you know, I mean
```

→ "Uh." / "Um, so." / "Yeah." 같은 블록이 단독으로 있으면 삭제

## GPT-4o 번역 프롬프트 설계

### 시스템 프롬프트 (`prompts.py`)

> "너는 전문 영상 자막 번역가이다"

#### 12가지 번역 규칙

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 번역 범위 제한 | `translate` 배열만 번역. `context_before`/`context_after`는 문맥 참고용 |
| 2 | id 정합성 | 출력 id 집합·순서 = 입력과 정확히 일치. id 추가/삭제/변경 금지 |
| 3 | 블록 경계 유지 | 한 블록 = 하나의 번역 (블록 분할/병합 금지) |
| 4 | 의역 우선 | 직역보다 의미 기반 자연스러운 표현. 정보 임의 추가/생략 금지 |
| 5 | 경어체 | 회의/교육 톤: ~합니다 / ~입니다 |
| 6 | 기술 용어 영문 유지 | Anzo, Ontology, RDF, SPARQL, API, SDK, JSON 등 |
| 7 | 고유명사 원문 유지 | 제품명, 버전명 등 |
| 8 | 서식 유지 | 이모티콘, 따옴표, 마크다운 등 의미 있을 때만 |
| 9 | 구어체 필러 제거 | uh, um, you know, like 등 자연스러운 범위 내 제거 |
| 10 | 줄바꿈 제한 | 원문 구조가 자연스러운 경우에만 사용 |
| **11** | **부호 구조 유지 (핵심)** | **원문이 쉼표(,)로 이어지면 번역도 쉼표로 유지. 원문에 마침표 없으면 번역에도 마침표 추가 금지** |
| 12 | JSON 스키마 준수 | 출력은 반드시 `{items:[{id, text}]}` 형식 |

#### 규칙 11이 중요한 이유

Whisper가 한 문장을 여러 블록으로 쪼갤 때, 각 블록을 독립 번역하면 GPT가 쉼표 문장을 마침표로 끊어버림.
→ 규칙 11로 원문의 부호 구조를 강제 유지하여 분할된 블록도 자연스럽게 이어지도록 함.

### 요청 페이로드 구조

```json
{
  "target_language": "Korean",
  "context_before": [
    {"id": 8, "text": "I gathered a few resources."}
  ],
  "translate": [
    {"id": 9, "text": "I put all of these in the chat."},
    {"id": 10, "text": "The first is what I already put in the chat, which is this is our service desk."}
  ],
  "context_after": [
    {"id": 11, "text": "In order to basically create any sort of ticket..."}
  ],
  "instruction": "translate 배열의 각 블록을 target_language로 번역하고, {items:[{id,text}]} 형태의 JSON으로만 응답하라."
}
```

- **context_before/after**: 앞뒤 5블록을 문맥으로 제공 (번역 대상 아님)
- **translate**: 실제 번역 대상 (30블록씩 chunk)
- **JSON Schema strict 모드**: GPT 응답이 반드시 `{items:[{id, text}]}` 형태를 따르도록 강제

### 용어집 (`glossary.json`)

```json
{
  "keep_as_is": ["Anzo", "Ontology", "SPARQL", "RDF", "Graph", "Cambridge Semantics"],
  "fixed_translations": {
    "knowledge graph": "지식 그래프",
    "data fabric": "데이터 패브릭"
  }
}
```

- `keep_as_is`: 번역하지 않고 영문 그대로 유지할 용어
- `fixed_translations`: 항상 지정된 번역을 사용할 매핑

### 오류 복구 전략 (`translator.py`)

```
1차 시도: 30블록 청크 전체 번역 요청
  ↓ id 누락 발견 시
2차 시도: 누락된 id만 재요청 (작은 청크)
  ↓ 그래도 실패 시
최종: 원문(영어) 유지 (번역 실패해도 자막 자체는 깨지지 않음)
```

## 기술 스택 요약

| 구성 요소 | 기술 | 역할 |
|-----------|------|------|
| 전사 (STT) | OpenAI Whisper API | 영상/오디오 → 영어 SRT |
| 번역 | GPT-4o + JSON Schema | 영어 → 한국어 자막 |
| 오디오 추출 | FFmpeg (imageio-ffmpeg) | 영상에서 오디오 분리, 25MB 분할 |
| 후처리 | Python (pysrt) | 블록 분할/병합/필러 제거 |
| 웹 GUI | Flask | 로컬 브라우저 기반 UI |
| 배포 | 임베디드 Python + 오프라인 wheels | 설치 없이 더블클릭 실행 |
