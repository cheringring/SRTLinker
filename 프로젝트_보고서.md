# SRTLinker 프로젝트 기술 보고서

## 1. 프로젝트 개요

**SRTLinker**는 영상/오디오 파일을 자동으로 전사(Whisper API)하고 한국어로 번역(GPT API)하여 자막 파일(SRT)을 생성하는 자동화 도구입니다. 웹 기반 GUI를 통해 드래그앤드롭 방식으로 손쉽게 사용할 수 있습니다.

---

## 2. 출력 파일 구조

### 2.1 영어 자막 저장 경로
```
output/
  └── en/
      ├── raw/
      │   └── 영상.raw.srt    ← Whisper 전사 원본 (한 번만 생성, 이후 재사용)
      └── 영상.en.srt          ← 정리된 영어 SRT (문장 병합 + 화자 분리)
```

**영어 자막 특징:**
- **raw 폴더**: Whisper API가 생성한 원본 전사 결과를 보존합니다. 한 번 생성되면 재사용되어 전사 비용이 중복으로 발생하지 않습니다.
- **en 폴더**: 후처리를 거친 정리된 영어 자막입니다. 코드 수정 시 raw 파일을 기반으로 매번 새로 생성됩니다.

**영어 자막 후처리 과정:**
1. **긴 블록 분할**: 8초 초과 블록을 문장 단위(`.`, `?`, `!`)로 자동 분할
2. **필러 제거**: "Uh.", "Um.", "Hmm." 등 의미 없는 필러 전용 블록 삭제
3. **화자 전환 감지**: 질문(`?`) 뒤 답변 패턴, 시간 갭(1초 이상), 응답 시작 패턴("Yeah", "Sure", "I think" 등)으로 화자 분리
4. **불완전 문장 병합**: 전치사/접속사로 끝나는 블록을 다음 블록에 자동 병합
5. **문장 병합**: 한 블록 = 한 문장으로 정리하여 번역 품질 향상

**결과:** 영어 자막은 완벽한 문장 구조로 정리되어 있으며, 화자 전환이 명확하게 분리되어 있습니다.

### 2.2 한국어 자막 저장 경로
```
output/
  └── ko/
      └── 영상.ko.srt          ← 한국어 번역 SRT
```

**한국어 자막 특징:**
- 영어 자막과 **1:1 블록 매핑** 유지 (동일한 블록 수, 동일한 타임스탬프)
- 정리된 영어 자막(`영상.en.srt`)을 기반으로 번역

## 3. 번역 품질 현황

### 3.1 기술 용어 처리 (✅ 완료)

**프롬프트 적용 규칙:**
- 기술 용어(Anzo, Ontology, Dataset, Knowledge Graph, RDF, SPARQL, API, SDK, JSON 등)는 영문 그대로 유지
- 고유명사/제품명/버전명은 원문 표기 유지
- `glossary.json` 파일을 통한 고정 번역 적용

**glossary.json 구조:**
```json
{
  "keep_as_is": ["Anzo", "SPARQL", "RDF", "Elasticsearch"],
  "fixed_translations": {
    "Knowledge Graph": "Knowledge Graph",
    "data fabric": "데이터 패브릭"
  }
}
```

**결과:** 기술 용어는 프롬프트에 명시된 대로 정확하게 반영되고 있습니다.

### 3.2 번역 문맥 보존 (⚠️ 개선 중)

**현재 상태:**
- 영어 자막은 완벽한 문장 구조로 정리됨
- 한국어 번역 과정에서 일부 문맥이 다르게 해석되는 경우 발생

**원인 분석:**
1. **LLM의 의역 특성**: GPT는 직역보다 의미 기반 자연스러운 표현을 우선하도록 설정되어 있어, 원문의 뉘앙스가 일부 변경될 수 있음
2. **문장 병합 과정**: 여러 블록을 하나의 문장으로 병합하는 과정에서 문맥 정보가 일부 손실될 수 있음
3. **슬라이딩 윈도우 한계**: 앞뒤 5개 블록을 문맥으로 제공하지만, 긴 대화나 복잡한 논리 전개에서는 충분하지 않을 수 있음

**적용된 개선 사항:**
- 슬라이딩 윈도우 방식으로 앞뒤 5개 블록을 문맥(`context_before`, `context_after`)으로 제공
- 원문 부호 구조 유지 규칙: "원문이 쉼표(,)로 이어지면 번역문도 쉼표로 유지, 마침표 임의 추가 금지"
- 회의/교육 톤에 맞는 경어 반말(~합니다/~입니다) 사용 지시

**향후 개선 방향:**
- 문맥 윈도우 크기 조정 실험 (현재 5개 → 7~10개)
- 프롬프트 규칙 세밀화 (특정 패턴에 대한 번역 예시 추가)
- 번역 후 검증 단계 추가 (원문과 번역문의 의미 일치도 체크)

**예상 소요 시간:** 프롬프트 최적화 및 검증 로직 추가에 약 2~3주 소요 예상

## 4. 프롬프팅 전략 및 구현

### 4.1 시스템 프롬프트 설계

GPT에게 "전문 영상 자막 번역가" 역할을 부여하고, 12가지 핵심 규칙을 시스템 프롬프트로 전달합니다.

**핵심 규칙 (`prompts.py`):**

1. **입출력 구조 규칙**
   - `translate` 배열의 각 블록만 번역 (`context_before/after`는 참고용)
   - 출력의 id 집합과 순서는 입력과 정확히 일치 (id 추가/삭제/변경 금지)
   - 한 블록 = 하나의 번역 (블록 경계 변경 불가)

2. **번역 품질 규칙**
   - 의역 허용, 직역보다 의미 기반 자연스러운 표현 우선
   - 정보를 임의 추가/생략하지 않음
   - 회의/교육 톤에 맞는 경어 반말(~합니다/~입니다) 사용

3. **용어 처리 규칙**
   - 기술 용어는 영문 그대로 유지
   - 고유명사/제품명/버전명은 원문 표기 유지
   - `glossary.json`에 정의된 고정 번역 반드시 적용

4. **부호 구조 유지 규칙 (핵심)**
   - 원문이 쉼표(,)로 이어지는 문장이면 번역문도 쉼표로 유지
   - 원문에 마침표가 없는데 번역문에 마침표를 추가하지 않음
   - 원문의 문장 부호 구조를 그대로 따름

### 4.2 번역 요청 페이로드 구조

```json
{
  "target_language": "Korean",
  "context_before": [{"id": 8, "text": "이전 블록 텍스트"}],
  "translate": [
    {"id": 9, "text": "번역할 블록 1"},
    {"id": 10, "text": "번역할 블록 2"}
  ],
  "context_after": [{"id": 11, "text": "다음 블록 텍스트"}],
  "instruction": "translate 배열의 각 블록을 target_language로 번역하고, {items:[{id,text}]} 형태의 JSON으로만 응답하라."
}
```

**특징:**
- 30개 블록씩 청크로 묶어서 번역 (병렬 처리 가능)
- 앞뒤 5개 블록을 문맥으로 제공하여 자연스러운 번역 유도
- OpenAI Structured Outputs (JSON Schema strict 모드)로 응답 형식 강제

### 4.3 번역 안정성 보장 메커니즘

**구현 위치:** `translator.py`

1. **JSON Schema Strict 모드**
   - GPT 응답을 `{items:[{id, text}]}` 형식으로 강제
   - id 매핑 오류 방지

2. **자동 재시도 로직**
   - id 누락 시 해당 블록만 자동 재요청 (최대 1회)
   - 최종적으로도 누락되면 원문 텍스트 유지 (빈 번역 방지)

3. **API 장애 대응**
   - Exponential backoff 재시도 (tenacity 라이브러리)
   - 일시적 네트워크 오류에 자동 복구

4. **텍스트 정리**
   - 줄바꿈/연속 공백 제거
   - 숫자만 있는 라인 제거 (GPT가 블록 번호를 에코하는 오류 방지)

## 5. 파이프라인 아키텍처

### 5.1 전체 처리 흐름 (`pipeline.py`)

```
입력 파일 (영상/오디오/SRT)
  │
  ├─ [영상/오디오] Whisper API 전사 → raw SRT 생성
  │   └─ output/en/raw/영상.raw.srt (한 번만, 이후 재사용)
  │
  ├─ 영어 SRT 후처리 (transcriber.py)
  │   ├─ 긴 블록 분할 (8초 초과 → 문장 단위로 분할)
  │   ├─ 필러 제거 ("Uh.", "Um." 등 필러 전용 블록 삭제)
  │   ├─ 불완전 문장 병합 (전치사/접속사로 끝나는 블록 → 다음 블록에 합침)
  │   ├─ 화자 전환 분리 (? 뒤 답변 패턴, 시간 갭, 응답 시작 패턴)
  │   └─ 문장 병합 (sentence_merger.py)
  │       └─ output/en/영상.en.srt
  │
  ├─ 번역 (translator.py)
  │   ├─ 30개 블록씩 청크로 묶어서 문맥(context_before/after) 제공
  │   ├─ JSON Schema strict 모드로 id 매핑 보장
  │   ├─ 원문 부호 구조 유지 (쉼표→마침표 변환 금지)
  │   └─ id 누락 시 해당 청크만 자동 재시도
  │       └─ output/ko/영상.ko.srt
  │
  └─ 영어/한국어 SRT 동일한 블록 수·타임스탬프 유지
```

### 5.2 주요 모듈 설명

**1. transcriber.py - 전사 및 영어 후처리**
- Whisper API 호출 (25MB 초과 시 10분 단위 자동 분할)
- 긴 블록 분할 (`_split_long_blocks`)
- 필러 제거 및 불완전 문장 병합 (`_clean_whisper_artifacts`)
- 화자 전환 감지 (질문→답변 패턴, 시간 갭, 응답 시작 패턴)

**2. sentence_merger.py - 문장 단위 병합**
- 연속 블록을 문장 단위로 그룹화 (`group_sentences`)
- 화자 전환 감지 (시간 갭 1초 이상, 질문 뒤 답변, 응답 시작 패턴)
- 병합된 SRT 생성 (`build_merged_srt`)

**3. translator.py - 번역 엔진**
- OpenAI API 호출 (Structured Outputs)
- 청크 단위 번역 (`translate_chunk`)
- id 누락 시 자동 재시도
- 텍스트 정리 (`_clean_translation`)

**4. prompts.py - 프롬프트 관리**
- 시스템 프롬프트 생성 (`build_system_prompt`)
- 용어집 자동 주입
- 사용자 페이로드 생성 (`build_user_payload`)

**5. pipeline.py - 전체 오케스트레이션**
- 파일 타입 감지 및 전사 여부 결정
- 영어 후처리 → 문장 병합 → 번역 순차 실행
- 진행률 콜백 제공 (웹 GUI 연동)

```

├── install.bat              # 자동 설치 스크립트 (Windows)

├── gui_web.py               # Flask 웹 GUI (localhost:8456)

├── srt_chunker.py           # SRT 파싱 / 청킹 / 재조립
├── prompts.py               # 번역 프롬프트 (규칙 + 용어집)
├── verify.py                # 번역 결과 정합성 검증
├── glossary.json            # 고유명사 & 고정 번역
├── test_translate.py        # 번역 퀵 테스트
├── requirements.txt         # 패키지 목록
├── README.md                # 상세 사용 설명서
├── 시작하기.md              # 빠른 시작 가이드
└── output/
    ├── en/
    │   ├── raw/             # Whisper 전사 원본 (재사용)
    │   └── *.en.srt         # 정리된 영어 SRT
    └── ko/
        └── *.ko.srt         # 한국어 번역 SRT
```

## 7. 기술 스택

- **언어**: Python 3.10+
- **전사**: OpenAI Whisper API
- **번역**: OpenAI GPT-4o (Structured Outputs)
- **웹 GUI**: Flask (localhost:8456)
- **오디오 처리**: imageio-ffmpeg (포터블 ffmpeg)
- **자막 처리**: pysrt
- **재시도 로직**: tenacity
- **환경 변수**: python-dotenv

---

## 8. 사용 비용

- **전사 비용**: Whisper API 사용료 (1시간 영상 기준 약 $0.3~$0.5)
- **번역 비용**: GPT-4o API 사용료 (1시간 영상 기준 약 $0.5~$2)
- **총 예상 비용**: 1시간 영상 기준 약 $0.8~$2.5

**비용 절감 메커니즘:**
- raw 전사 결과 재사용 (코드 수정 시 전사 비용 재발생 안 함)
- 병렬 처리로 API 호출 최소화
- 청크 단위 번역으로 효율성 향상

## 10. 결론

**SRTLinker**는 영상 자막 자동 생성 및 번역을 위한 완성도 높은 도구입니다.

**강점:**
- ✅ 영어 자막은 완벽한 문장 구조로 정리됨
- ✅ 기술 용어는 프롬프트 규칙에 따라 정확하게 반영됨
- ✅ 1:1 블록 매핑으로 타임스탬프 정합성 보장
- ✅ 자동 재시도 및 오류 복구 메커니즘 완비
- ✅ 웹 GUI로 손쉬운 사용 가능

**개선 필요 사항:**
- ⚠️ 한국어 번역 과정에서 일부 문맥이 다르게 해석되는 경우 발생
- ⚠️ LLM의 의역 특성으로 인한 뉘앙스 변화
- ⚠️ 완벽한 번역을 위해서는 프롬프트 최적화 및 검증 로직 추가 필요 (예상 소요: 2~3주)

**현재 상태:**
- 실무 사용 가능한 수준의 번역 품질 제공
- 기술 용어 및 고유명사는 정확하게 처리됨
- 일반적인 회의/교육 영상에 적합
